## 🏙️ 산업 및 서비스 특화 분야의 주요 컴플라이언스 프레임워크

---

## ✅ 1. **TISAX (Trusted Information Security Assessment Exchange)**

| 항목        | 설명                                                                                 |
| --------- | ---------------------------------------------------------------------------------- |
| **목적**    | 자동차 산업 내 **정보보안 수준의 표준화 및 상호 인증**                                                  |
| **적용 산업** | **자동차 산업** (OEM, 부품사, 공급망 전체)                                                      |
| **주요 내용** | ISO/IEC 27001 기반 보안 요구사항 + **산업 맞춤 보완 기준**<br>보안, **프로토타입 보호**, **데이터 보호(GDPR)** 등 |
| **특징**    | 독일 자동차 산업 협회(VDA) 주도, **TISAX 포털을 통한 상호 인증 공유**                                    |

---

## ✅ 2. **CSMS (Cyber Security Management System)**

| 항목        | 설명                                                                                             |
| --------- | ---------------------------------------------------------------------------------------------- |
| **목적**    | 자동차에 대한 **사이버 보안 리스크 관리 체계 수립**                                                                |
| **적용 산업** | **자동차 제조 및 전장 시스템 개발**                                                                         |
| **주요 내용** | **UNECE WP.29** (R155) 규정에 기반하여 차량 개발 전 주기(개발-생산-운용-폐기)에서 보안 관리 체계 요구<br>위협 식별, 위험 분석, 보안 조치 등 |
| **특징**    | CSMS 인증 없이는 **EU 및 UN 규정 적용 국가에서 차량 판매 불가**                                                    |

---

## ✅ 3. **CSAP (Cloud Security Assurance Program)**

| 항목        | 설명                                                                     |
| --------- | ---------------------------------------------------------------------- |
| **목적**    | 클라우드 서비스의 **보안 수준을 평가**하고 이용자 신뢰 확보                                    |
| **적용 산업** | **클라우드 서비스 제공자(CSP)** 및 **공공·민간 클라우드 도입 기업**                           |
| **주요 내용** | 국내에서는 **과학기술정보통신부 / KISA** 주관<br>보안 수준을 **1\~3등급**으로 평가, 보안통제 항목 기준 제공 |
| **특징**    | **공공기관의 클라우드 도입 시 필수**, 민간에서도 점차 확대 적용                                 |

---

## ✅ 4. **SOC 2 (System and Organization Controls Type 2)**

| 항목        | 설명                                                                              |
| --------- | ------------------------------------------------------------------------------- |
| **목적**    | **클라우드 기반 서비스** 제공자의 **보안·신뢰성 통제 검증**                                           |
| **적용 산업** | **SaaS, PaaS 등 클라우드 기반 서비스 제공 기업**                                              |
| **주요 내용** | AICPA(미국 공인회계사협회) 기준<br>보안, 가용성, 기밀성, 무결성, 개인정보보호 등 **5대 신뢰 서비스 원칙(TSC)** 기반 평가 |
| **특징**    | 특히 **미국 및 글로벌 B2B 고객과 계약 시 신뢰 확보 수단**으로 널리 사용                                   |

---

## ✅ 5. **CC (Common Criteria, 공통평가기준)**

| 항목        | 설명                                                                                        |
| --------- | ----------------------------------------------------------------------------------------- |
| **목적**    | IT 제품 및 시스템의 **보안 기능을 국제적으로 검증**                                                          |
| **적용 산업** | **정보보안 솔루션 제조사** (예: 보안 SW, 인증 장치, OS 등)                                                  |
| **주요 내용** | 국제 표준 ISO/IEC 15408<br>보안 기능 요구사항 및 보증 요구사항으로 나뉨<br>평가보증 등급(EAL 1\~7)을 통해 **보안 신뢰 수준 표시** |
| **특징**    | \*\*국가 간 상호인정협정(CCRA)\*\*을 통해 **글로벌 제품 판매에 활용 가능**                                        |

---

## 🧩 요약 비교표

| 구분        | 대상/산업        | 목적            | 주요 내용                |
| --------- | ------------ | ------------- | -------------------- |
| **TISAX** | 자동차 공급망      | 정보보안 인증 공유    | ISO 27001 기반 + 산업 특화 |
| **CSMS**  | 차량 사이버보안     | 전 주기 사이버보안 체계 | UNECE R155 기반        |
| **CSAP**  | 클라우드 보안      | 보안 수준 인증      | 등급별 평가 (KISA 등)      |
| **SOC 2** | 클라우드/SaaS 기업 | 신뢰성 통제 검증     | 5대 신뢰 원칙 평가          |
| **CC**    | 보안 제품        | 보안 기능 국제 인증   | EAL 등급화, CCRA 상호인정   |
