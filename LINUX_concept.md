#### **사운드, 스캐너, 프린터**

* **OSS**: 리눅스/유닉스 계열 사운드 카드 지원 시스템
* **ALSA (Advanced Linux Sound Architecture)** ✅: 사운드 카드용 장치 드라이버, 사운드 장치를 관리
* **alsactl**: 사운드 카드를 제어하는 명령어
* **SANE** ✅: 스캐너, 비디오 카메라 등의 이미지 관련 API (GTK 라이브러리와 연관, XSANE 사용)
* **CUPS**: 애플이 개발한 프린팅 시스템
* **LPRng**: BSD 계열의 프린팅 시스템

* **System V**:

  * `Ip`: 프린터 작업 요청

* **IPP (Internet Printing Protocol)**: 인터넷 원격 프린팅

#### **디스크 관리 및 RAID**

* **Mirroring**: 디스크에 에러 발생 시 데이터 손실 방지를 위해 추가 장치에 중복 저장
* **Volume Group (VG)**: 물리적 볼륨(PV)을 묶어 하나의 그룹으로 생성

* **LVM (Logical Volume Manager)**: 리눅스의 저장 공간을 효율적으로 관리하는 커널의 부분

  * **PE (Physical Extent)**: PV에서 나누어 사용하는 블록 (4MB 할당) ✅
  * **PV (Physical Volume)**: LVM에서 사용할 수 있도록 디스크 변환
  * **VG (Volume Group)**: PV들이 묶여 만들어진 그룹
  * **LV (Logical Volume)**: VG에서 할당된 파티션
  * **LE (Logical Extent)**: LV 내의 블록, PE와 1:1 대응

# vi 편집기

## 개요

* 유닉스 계열 시스템에서 가장 많이 사용되는 편집기.
* 1976년 빌 조이에 의해 개발됨.

---

## ex 모드 명령어

* **`%d`** : 파일 안의 모든 내용을 지우기 위한 ex 모드 명령어.

* **`%s/windows/linux/g`** : "windows" 문자열을 모두 "linux"로 치환.

* **예시**:

  * `1,7s/fail/success/g` : 1번부터 7번까지의 줄에서 "fail"을 "success"로 치환.
  * `%s/^linux/Linux/` : 파일 전체에서 "linux"로 시작하는 줄을 "Linux"로 치환.
  * ` %s/￦linux/Linux/` : 줄의 끝부분에서 "linux"를 "Linux"로 치환.
  * ` %s/<\linux>/Linux/` : "linux"라는 단어를 "Linux"로 치환.

* **예시 2**:

  * `1,9 %s/문자열/변경할 문자열` : 전체 9줄을 "변경할 문자열"로 치환.

---

## 자동 치환 설정

* **`ab uz uzoogom`** : "uz"라는 문자열을 입력 시 자동으로 "uzoogom"으로 치환되도록 설정.

  * 이 설정은 `.exrc` 파일에 추가해야 함.

---

## vi 편집기 설정 파일

* **`.exrc`** : vi 편집기의 환경 설정을 등록하여, 실행 시 계속적으로 지정된 설정을 적용.
* **`.vimrc`** : vi 편집기 실행 시, 자동으로 행 번호를 표시하도록 설정하는 파일.

---

## vi 명령어 및 설정

* **`vi +15`** 또는 **`vi -ic 15`** : 파일을 불러올 때 커서를 15번째 줄로 이동.

  * 예시: `vi +line.txt` : 파일을 열면서 커서가 마지막 줄로 이동.

* **`set nu`** : 문서의 각 라인 번호를 표시.

  * 각 행 앞에 행 번호가 나타나도록 설정.
  * 일반적으로 `set` 명령을 사용할 때는 `:` (콜론)을 사용하지 않음.

* **`set ai`** : 위 줄과 같은 열에 커서를 위치할 때 지정하는 환경 설정.

* **`set sm`** : 괄호 강조.

* **`set ts`** : 탭 간격 조정.

* **`set list`** : 개행 문자 (`$`), TAB 문자 (`^|`) 등을 확인할 수 있도록 설정.

---

## 파일 복구

* **`r`** : 비정상적으로 종료된 후 작업 중이던 파일을 불러오는 명령.

  * `.swp` 파일의 내용을 불러옴.

---

## 편집 모드 전환

* **`l`, `o`, `a`** : 입력 모드로 전환하는 명령.

  * **`I`** : 커서 위치에서 편집 모드로 전환.
  * **`o`** : 다음 줄로 넘어가서 편집 모드로 전환.
  * **`a`** : 한 칸 뒤로 넘어가서 편집 모드로 전환.

---

## 커서 이동

* **`vi $ 파일명`** : 커서를 줄의 맨 끝으로 이동.
* **`vi - 파일명`** : 커서를 앞줄의 첫 문자로 이동.

##  vi 편집기

###  vi 모드

* **입력 모드 (Insert Mode)**: 텍스트를 입력할 수 있는 모드.
* **명령 모드 (Normal Mode)**: 기본적으로 vi를 실행할 때 시작되는 모드로, 커서 이동과 명령을 수행할 수 있음.
* **ex 모드 (Edit Mode)**: 파일을 저장하거나 종료하는 등의 명령을 입력하는 모드.
* **작업 모드**: 사용되지 않는 용어.

---

### vi 명령어

* **`3,9s/^#`**: 3번째 줄부터 9번째 줄까지 주석을 제거. (주석은 `#`으로 시작)

### 입력 모드 명령어

* **`a`**: 커서 뒤부터 입력.
* **`A`**: 줄의 마지막에 입력.

* **`o`**: 현재 줄의 다음 줄에 입력.
* **`O`**: 커서가 있는 줄 위에 입력.
* **`x`**: 커서 위치한 곳의 한 문자 삭제.
* **`dd`**: 한 줄 삭제.
* **`dw`**: 커서 위치에서 오른쪽 한 단어 삭제.
* **`db`**: 커서 위치에서 왼쪽 한 단어 삭제.
* **`h`**: 왼쪽으로 이동.
* **`j`**: 아래로 이동.
* **`k`**: 위로 이동.
* **`l`**: 오른쪽으로 이동.
* **`yy`**: 복사.
* **`p`**: 붙여넣기.

## vim 편집기

    * 편집 시 다양한 색상을 이용하여 가시성을 높일 수 있음.
    * 정규 표현식 문법과 강력한 문법 강조 기능 제공.

## pico 편집기

### 개요

* **pico** (by **아보일 카사르**): 간단하고 직관적인 텍스트 편집기.

## PICO 편집기

* **개요**:

  * 워싱턴 대학에서 만든 유닉스용 편집기.
  * 윈도우의 메모장처럼 간편하게 사용하도록 설계됨.
  * 모드가 따로 존재하지 않으며, 바로 텍스트 입력이 가능.

* **특징**:

  * **라이선스 문제**: 리눅스 초기에는 대부분의 배포판에 포함되어 있었지만, 최근에는 라이선스 문제로 인해 포함되지 않음.
  * **소스 수정 불가**: 자유 소프트웨어 라이선스가 아니어서 소스 코드 수정이 불가능.
  * **단축키 설명 표시**: 에디터 화면 아래에 단축키에 대한 설명이 표시됨.
  * **배포**: Pine이라는 E-mail 클라이언트 프로그램과 함께 배포됨.

* **주요 단축키**:

  * **`Ctrl + k`**: 한 줄 삭제.
  * **`Ctrl + u`**: 붙여넣기.
  * **`Ctrl + o`**: 편집된 내용 저장.
  * **`Ctrl + a`**: 커서를 현재 줄의 시작 부분으로 이동 (반대는 `Ctrl + e`).
  * **`Ctrl + x`**: 종료.

## Nano 편집기

* **특징**:

  * **다양한 기능 제공**: C, LISP, FORTRAN, HTML 등의 소스 코드 편집을 위한 모드를 지원.

## 프로세스 관리

* **`nice`**:

    \*\*`nice`\*\*의 **양수값**은 **우선순위를 낮추고**, **음수값**은 **우선순위를 높입니다**.

# 리눅스 명령어 및 시스템 호출

## 프로세스 관리 명령어

* **`nice`**
  프로세스 우선순위를 설정하는 명령어.

  * **기본값**: `nice` 명령어에서 `-n` 값을 생략하면 기본값은 **10**입니다.
  * **예시**: `nice --10 bash`
    `bash` 프로세스의 **NI 값**을 10만큼 감소시켜 **우선순위를 높임**.

* **`renice`**
  실행 중인 프로세스의 우선순위를 변경하는 명령어.

  * **옵션**: `renice n PID`

    * `n`: 우선순위 변경 값
    * `PID`: 프로세스 ID
  * **우선순위 변경 규칙**:

    * **양수값**: 우선순위를 **낮춤**.
    * **음수값**: 우선순위를 **높임**.

* **`pstree`**
  프로세스를 **트리 형식**으로 출력하여 부모-자식 관계를 시각적으로 확인할 수 있는 명령어.

* **`jobs`**
  현재 **백그라운드 프로세스** 및 작업 상태를 확인할 때 사용하는 명령어.

  * **상태**:

    * 작업이 **중지된 상태**
    * **백그라운드로 실행 중인 상태**
    * **변경되었지만 보고되지 않은 상태**

* **`nohup`**
  터미널이 닫혀도 실행 중인 프로세스를 **백그라운드 프로세스**로 실행할 때 사용.

  * **예시**: `nohup <command> &`

* **`bg`**
  현재 실행 중인 프로세스를 **백그라운드 작업**으로 전환.

  * **`fg`**: 백그라운드 작업을 **포그라운드**로 전환.
  * **예시**: `bg 1`, `fg 1`: `1`번 작업을 백그라운드 또는 포그라운드로 전환.

* **`&`**
  명령어 뒤에 `&`를 붙이면 **백그라운드**에서 작업을 실행.

  * **예시**: `bash script.sh &`: `script.sh`를 백그라운드에서 실행.

# 리눅스 데몬, CRON, 시그널

## 데몬 (Daemon)

* **데몬**: 리눅스 시스템이 부팅 시 자동으로 실행되는 **백그라운드 프로세스**.

  * **주요 특징**:

    * 주기적이고 지속적인 **서비스 요청 처리**를 위해 항상 실행됩니다.
    * 일반 사용자는 이 프로세스를 볼 수 없으며, **루트 권한**만 필요합니다.
  * **종류**:

    * **Standalone**: 메모리에 항상 상주하여 실행되는 방식.

      * **단점**: 메모리 점유로 인한 서버 부하가 큼.
    * **항상 상주하지 않음**: 요청이 들어오면 관련 프로세스를 실행하고 요청이 끝나면 종료됩니다.
* **부팅 시 데몬 실행 방식**:

  * **CentOS 7 이전**: `init` 시스템
  * **CentOS 7 이후**: `systemd` 시스템

## CRON (주기적 작업 자동화)

* **cron**: 주기적으로 반복되는 일을 **자동적으로 실행**하는 서비스.

  * **crontab**: 사용자의 **개별 crontab 파일**을 관리하는 명령어.
* **주요 명령어**:

  * `crontab -e`: **crontab**을 작성하거나 수정
  * `crontab -u <user> -e`: 특정 사용자의 **crontab**을 수정
  * `crontab -r`: 사용자의 **crontab** 삭제
  * `crontab -l`: 현재 사용자의 **crontab 내용 출력**
* **CRON 날짜 형식** (순서):

  ```
  분   // 시   // 일   // 월   // 요일
  ```

  * **요일**: `일(0), 월(1), 화(2), 수(3), 목(4), 금(5), 토(6)` → 0부터 6까지 번호로 지정.
* **주기적인 작업 스케줄링 명령어**:

  * `crontab`, `at`: 주기적인 작업이나 지정된 시간에 실행되는 작업을 설정하는 명령어.

## Kill 시그널

* **시그널 (Signal)**: 프로세스에 특정 작업을 요청하는 메커니즘.

  * **SIGHUP**:

    * **기능**: 프로세스를 **재시작**하거나 터미널과의 연결이 끊겼을 때 발생.
    * **시그널 번호**: 1

  * **SIGINT**:

    * **기능**: 프로세스를 **종료**시키며, 보통 **Ctrl+C**로 발생.
    * **시그널 번호**: 2

# 시그널, 우선순위 및 환경설정 파일

## 시그널 (Signal)

* **SIGQUIT**:

  * **기능**: 프로세스를 종료하고 **코어 덤프**를 생성.
  * **기본 동작**: `Ctrl + \`
  * **시그널 번호**: 3

* **SIGKILL**:

  * **기능**: 프로세스를 **강제로 종료**.
  * **기본 동작**: `kill -9`
  * **시그널 번호**: 9

* **SIGTERM**:

  * **기능**: **정상 종료**를 시도하는 시그널.
  * **기본 동작**: `kill` 명령어로 보내는 기본 시그널
  * **시그널 번호**: 15

* **SIGTSTP**:

  * **기능**: 프로세스를 **대기 상태로 전환**.
  * **기본 동작**: `Ctrl + Z`
  * **시그널 번호**: 20

* **예시**:

  * `kill -1 %2`: **jobs** 명령으로 출력되는 2번 작업에 `SIGHUP` 시그널을 보내어 종료 신호를 보냄.

* **killall**:

  * **기능**: PID 대신 **프로세스명**을 사용하여 해당 프로세스를 종료.

## 프로세스 우선순위

* **PRI (Priority)**:

  * **설명**: 커널이 사용하는 프로세스의 우선순위. 운영체제가 실제로 참고하는 우선순위 값.

* **NI (Nice Value)**:

  * **설명**: 사용자가 설정할 수 있는 프로세스의 우선순위 항목.
  * **효과**: NI 값이 PRI 값에 영향을 미침.

* **우선순위 확인 명령어**:

  * `ps`: 현재 로그인에서 사용 중인 프로세스의 NI 및 PRI 값을 확인할 수 있는 명령어.

## 환경설정 파일

* **`~/.bashrc`**:

  * **설명**: **개인 사용자 환경 설정** 파일. 주로 alias, 함수 등을 설정.

* **`~/.bash_profile`**:

  * **설명**: **로그인 쉘 환경 설정** 파일. 주로 환경 변수, 프로그램 실행 등을 설정.

* **`/etc/bashrc`**:

  * **설명**: **시스템 전체의 bash 설정** 파일. 전역적인 환경 설정.

* **`/etc/shells`**:

  * **설명**: 리눅스 시스템에서 사용 가능한 **셸 정보**가 저장된 파일.

* **`/etc/passwd`**:

  * **설명**: 시스템의 **사용자 정보**가 저장된 파일. 사용자의 계정 정보, UID, GID 등이 포함됨.

## 리눅스 시스템 파일 및 설정

### 로그인 셸 확인

* **명령어**:
  `echo $SHELL`

  * **기능**: 현재 로그인한 사용자의 기본 셸을 확인.

### 시스템 사용자 정보 관련 파일

* **`/etc/shadow`**:

  * **설명**: 시스템 사용자들의 **비밀번호 정보 및 비밀번호 정책**을 저장.
  * **특징**: 일반 사용자는 이 파일을 **볼 수 없음**.

* **`/etc/group`**:

  * **설명**: **로컬 그룹에 대한 정보** 저장. 각 그룹의 이름, GID, 사용자 목록 등.

### 파일시스템 정보 관련 파일

* **`/etc/fstab`**:

  * **설명**: 시스템의 **파일시스템 정보**를 저장.
  * **구성**: 6개의 필드로 구성되어 있음.

    1. **파일시스템 장치**
    2. **마운트 포인트**
    3. **파일시스템 종류**
    4. **옵션**
    5. **덤프** (백업 유무)
    6. **파일점검 옵션** (fsck와 관련)
  * **옵션**:

    * `a`: 파일 시스템을 마운트할 때 사용하는 옵션.
    * `user`: 일반 사용자가 파일 시스템을 마운트할 수 있는 권한 부여.
    *  파일 마운트란 파일 파티션에 디렉토리 형태로 접근하는 것을 의미
* **디스크 쿼터 설정 옵션**:

  * **옵션**:

    * `quota`, `uquota`, `usrquota`, `userquota` 등의 값을 사용하여 디스크 쿼터 설정.

### 부팅 및 환경 설정 파일

* **`/etc/inittab`**:

  * **설명**: 시스템 **부팅 과정 및 초기화 설정**에 관련된 파일.
  * **특징**: 부팅 시 **기본적인 실행 프로세스** 및 X 윈도우 설정을 관리.

* **`/etc/profile`**:

  * **설명**: 시스템의 **환경 변수**와 **시작 프로그램**을 설정하는 파일.
  * **특징**: 모든 사용자에게 반영되는 시스템 전체 설정 파일.

* **`/etc/profile.d`**:

  * **설명**: 여러 응용 프로그램들이 시작될 때 필요한 **스크립트**들이 저장되는 디렉터리.
  * **특징**: 시스템 전역의 환경 변수 설정을 보완하는 스크립트들.

* **`/etc/hosts`**:

  * **설명**: **IP 주소**와 **도메인 이름**을 매핑하는 파일.
  * **기능**: 네트워크에서 도메인 이름을 IP 주소로 변환하는 로컬 설정을 포함.

* **`/etc/sysconfig/network-scripts`**:

  * **설명**: **네트워크 인터페이스 환경 설정**과 관련된 파일들이 저장된 디렉터리.
  * **파일 예시**:

    * `ifcfg-eth0`: 이더넷 인터페이스의 설정 파일.
    * `ifcfg-lo`: 로컬 루프백 인터페이스 설정 파일.

* **`/etc/resolv.conf`**:

  * **설명**: **DNS 서버 주소**를 설정하는 파일.
  * **기능**: 시스템이 DNS 요청을 보낼 서버의 IP 주소를 지정.

* **`/etc/services`**:

  * **설명**: **프로토콜의 포트번호**를 확인할 수 있는 파일.
  * **기능**: TCP, UDP와 같은 프로토콜별로 사용되는 포트 번호와 서비스 이름을 매핑.

### 사용자 설정 관련 명령어

* **`chsh`** (Change Shell):

  * **기능**: 사용자의 로그인 셸을 변경할 때 사용.
  * **옵션**:

    1. `chsh -l`: 변경 가능한 쉘 목록을 출력.
    2. `chsh -s [쉘] [사용자명]`: 사용자의 로그인 쉘을 변경.
       예) `chsh -s /bin/bash username`
    3. `chsh -u`: `chsh` 명령어에 대한 **도움말** 출력.

### 쉘 종류

* **`csh` (C Shell)**:

  * **기원**: 버클리 대학에서 개발.
  * **특징**: C 언어를 기반으로 한 쉘. 사용법이 C 언어와 유사.

* **`ksh` (Korn Shell)**:

  * **기원**: AT\&T의 데이비드 콘에 의해 개발.
  * **특징**: 본쉘(Bourne Shell)을 확장하여 개발되었으며, 많은 고급 기능을 제공.

* **`bash` (Bourne Again Shell)**:

  * **기원**: 본쉘(Bourne Shell)의 개선된 버전.
  * **특징**: 리눅스와 유닉스 시스템에서 가장 많이 사용되는 쉘로, 명령어 자동완성, 명령 히스토리 등 많은 기능을 제공.

## 쉘 종류 및 관련 명령어

### 쉘 종류

* **`csh` (C Shell)**:

  * **기원**: 1978년, 버클리 대학에서 개발.
  * **특징**: C 언어의 문법을 모방한 쉘.

* **`bash` (Bourne Again Shell)**:

  * **기원**: 본쉘(Bourne Shell)을 개선하여, GNU 프로젝트에 의해 개발.
  * **특징**: 가장 널리 사용되는 쉘로, 명령어 자동완성, 히스토리 기능 등 다양한 기능을 제공.

* **`tcsh` (TENEX C Shell)**:

  * **기원**: 1981년, 캔 그리어가 개발.
  * **특징**: C 쉘을 확장하여, 명령행 완성 기능을 통합.

* **`dash` (Debian Almquist Shell)**:

  * **기원**: 더 가볍고 빠른 쉘을 목표로 개발된 쉘.
  * **특징**: 시스템 스크립트에서 빠른 실행을 제공, `/bin/sh`의 대체 쉘로 사용.

### 가장 최근에 등장한 쉘은?

* **`dash`**:

  * **답**: \*\*가장 최근에 등장한 쉘은 `dash`\*\*입니다.

### 쉘 관련 명령어

* **`history`**:

  * **기능**: 이전에 실행한 명령어들을 출력하는 명령어.
  * **예시**: `history 5` → 최근에 실행한 5개의 명령어 출력.

* **`set`**:

  * **기능**: 셸 변수나 환경 변수를 확인하거나 설정할 때 사용.
  * **사용 예시**: 사용자가 로그인한 후 선언한 모든 셸 변수를 확인할 수 있음.

* **`eject`**:

  * **기능**: CD-ROM, DVD-ROM 등의 미디어 장치를 꺼내는 명령어.
  * **사용 예시**: `eject` → CD/DVD 드라이브의 미디어를 꺼냄.

* **`mount`**:

  * **기능**: 파일 시스템을 특정 디렉터리에 마운트하는 명령어.
  * **예시**:

    ```bash
    mount -t xfs /dev/sdb1 /backup
    ```

    * **설명**: `/dev/sdb1` 디바이스를 `/backup` 디렉터리에 XFS 파일 시스템으로 마운트.

* **`umount`**:

  * **기능**: 파일 시스템을 마운트 해제하는 명령어.
  * **예시**:

    ```bash
    umount -a
    ```

    * **설명**: `/etc/mtab`에 명시된 모든 파일 시스템을 마운트 해제. 만약 해제할 수 없는 경우 "device is busy" 메시지가 출력됨.

### 관련 파일

* **`/etc/mtab`**:

  * **기능**: 현재 마운트된 파일 시스템에 대한 정보를 저장하는 파일. `umount -a` 명령은 이 파일을 참조하여 마운트 해제할 파일 시스템을 결정.

## 파일 시스템 및 디바이스 관련 명령어

### `mke2fs`

* **기능**: 파일 시스템을 생성하고 포맷하는 명령어.
* **주요 옵션**:

  * `-t [파일시스템형식]`: 파일 시스템의 유형을 지정.
    예시: `mke2fs -t ext2 /dev/sdb1` → `/dev/sdb1` 디바이스를 ext2 파일 시스템으로 포맷.
  * `-j`: ext3 파일 시스템을 생성.
    예시: `mke2fs -j /dev/sdb1` → `/dev/sdb1`을 ext3 파일 시스템으로 포맷.

### `mkfs.xfs`

* **기능**: XFS 파일 시스템을 생성하는 명령어.
* **사용 예시**:

  ```bash
  mkfs.xfs /dev/sdb1
  ```

  → `/dev/sdb1`을 XFS 파일 시스템으로 포맷.

### 마운트 관련 필드

1. **File system**: 마운트할 파일 시스템.
2. **Mount point**: 파일 시스템이 마운트될 디렉터리.
3. **Type**: 파일 시스템의 종류 (예: ext4, xfs 등).
4. **Option**: 마운트 옵션 (읽기/쓰기 권한 등).
5. **Dump**: 덤프 유틸리티가 백업할지 여부를 설정하는 필드.
6. **Pass**: `fsck` 명령어로 파일 시스템 점검 시 점검 순서.

## 파일 허가권 및 접근 권한

### `chmod`

* **기능**: 파일이나 디렉터리의 접근 권한을 설정하는 명령어.

* **주요 권한**:

  * **소유자 (owner)**: `u`
  * **그룹 (group)**: `g`
  * **기타 사용자 (others)**: `o`
  * **모든 사용자 (all)**: `a`

* **권한 숫자 표현**:

  * `r` (읽기): 4
  * `w` (쓰기): 2
  * `x` (실행): 1

* **옵션**:

  * `-R`: 디렉터리와 하위 디렉터리까지 적용.

#### 예시:

* `chmod 777 /path/to/file`: 모든 사용자에게 읽기, 쓰기, 실행 권한을 부여.
* `chmod u+x file`: 파일 소유자에게 실행 권한을 추가.
* `chmod -R 755 /path/to/directory`: 해당 디렉터리와 하위 디렉터리들에 대해 `755` 권한을 설정.

* **`df`**: 파일 시스템의 디스크 공간 사용 상태를 확인.

### `chmod` (파일 권한 변경)

`chmod` 명령어는 파일이나 디렉터리의 권한을 설정하거나 변경하는 데 사용됩니다. 여기서는 다양한 옵션과 모드 설정 방법을 설명합니다.

#### 옵션

* **`-v`**: 파일 권한이 변경될 때마다 해당 변경 사항을 출력합니다 (diagnostic message).
* **`-f`**: 오류 메시지를 출력하지 않습니다 (Quiet 모드).
* **`-c`**: 기존 권한에서 변경된 경우에만 메시지를 출력합니다.
* **`-R`**: 디렉터리 및 하위 디렉터리까지 재귀적으로 권한을 적용합니다.

#### MODE (파일 권한 설정 방법)

* **`u`**: 파일 소유자 (user).
* **`g`**: 파일 소속 그룹 (group).
* **`o`**: 기타 사용자 (others).
* **`a`**: 모든 사용자 (all).

권한 부여 방법:

* **`+`**: 권한을 추가합니다 (예: `+x`는 실행 권한을 추가).
* **`-`**: 권한을 제거합니다 (예: `-x`는 실행 권한을 제거).
* **`=`**: 권한을 정확히 설정합니다 (예: `=rw`는 읽기/쓰기 권한을 설정).

권한 종류:

* **`r`**: 읽기 권한 (read).
* **`w`**: 쓰기 권한 (write).
* **`x`**: 실행 권한 (execute).
* **`X`**: 디렉토리나 이미 실행 권한이 있는 파일에만 실행 권한을 부여.
* **`s`**: Setuid, Setgid 권한 설정 (특수 권한).
* **`t`**: Sticky bit, 디렉토리에서 삭제 권한 제한.
* **`0~7`**: 8진수 모드 설정, 각 숫자는 파일 권한을 설정하는 값을 나타냅니다.

#### 예시:

* `chmod u+x file.txt`: 소유자에게 실행 권한을 추가.
* `chmod g-r file.txt`: 그룹에서 읽기 권한을 제거.
* `chmod o=r file.txt`: 기타 사용자에게 읽기 권한만 부여.
* `chmod 755 file.txt`: 소유자에게 읽기/쓰기/실행 권한을 부여하고, 그룹과 기타 사용자에게 읽기 및 실행 권한을 부여.

### `chown` (파일 소유자 변경)

`chown` 명령어는 파일이나 디렉터리의 소유자 및 그룹을 변경하는 데 사용됩니다.

#### 기본 사용법:

```bash
chown [소유자]:[그룹] [파일명]
```

* **소유자**: 파일의 소유자를 지정.
* **그룹**: 파일의 소속 그룹을 지정.

#### 예시:

* `chown user:group file.txt`: `file.txt`의 소유자를 `user`, 소속 그룹을 `group`으로 변경.
* `chown root file.txt`: `file.txt`의 소유자를 `root`로 변경 (그룹은 변경되지 않음).
* `chown -R user:group /path/to/dir`: 디렉터리 및 그 하위 파일들의 소유자와 그룹을 `user`와 `group`으로 변경 (재귀적 변경).

#### 옵션:

* **`-R`**: 하위 디렉터리 및 파일들까지 재귀적으로 소유자를 변경.

---

### `chgrp` (파일 그룹 변경)

`chgrp` 명령어는 파일이나 디렉터리의 소속 그룹을 변경하는 데 사용됩니다.

#### 기본 사용법:

```bash
chgrp [그룹] [파일명]
```

* **그룹**: 변경할 그룹의 이름.

#### 예시:

* `chgrp group file.txt`: `file.txt`의 그룹을 `group`으로 변경.
* `chgrp -h group symlink`: 심볼릭 링크의 그룹을 변경 (실제 파일이 아닌 링크에 대해 그룹을 변경).

### `umask` (파일 생성 시 기본 권한 설정)

`umask`는 새로 생성된 파일이나 디렉터리의 기본 권한을 설정합니다. 기본적으로 `umask`는 **제거할 권한**을 설정합니다.

#### 예시:

* `umask 022`: 새로 생성된 파일은 **644** 권한을 가짐 (소유자는 읽기/쓰기 권한, 그룹 및 기타 사용자는 읽기 권한).
* `umask 0777`: 새로 생성된 파일은 **000** 권한을 가짐 (파일을 생성할 수 없으며, 권한이 설정되지 않음).

#### `umask`의 동작 원리:

* 새로 생성된 파일의 권한에서 `umask` 값만큼 제거됩니다.

  * 예를 들어, 파일의 기본 권한이 666이고, `umask`가 022라면 생성되는 파일의 권한은 `644`가 됩니다.
  * 디렉터리의 기본 권한은 777이고, `umask`가 022라면 생성되는 디렉터리의 권한은 `755`가 됩니다.

---

### **`umask` - 기본 파일 권한 제한**

`umask`는 새로 생성된 파일이나 디렉터리에 대해 기본 권한을 제한하는 설정 값입니다. `umask`는 파일이나 디렉터리의 기본 권한에서 제거할 권한을 지정합니다.

#### **기본 권한:**

* **파일**: 666 (rw-rw-rw-)
* **디렉터리**: 777 (rwxrwxrwx)

#### **`umask` 값:**

* `umask` 값은 3자리 숫자로 구성되며, 각 자리는 사용자(user), 그룹(group), 기타 사용자(others)에 대해 권한을 설정합니다.
* 각 자리는 **읽기(r)**, **쓰기(w)**, **실행(x)** 권한을 나타내는 비트 조합으로, 권한을 **제거**하는 값입니다.

#### **예시:**

* `umask 022`: 기본 권한에서 **쓰기 권한**을 **제거**합니다.

  * 기본 파일 권한 `666`에서 `022`를 빼면, 파일의 권한은 `644` (rw-r--r--).
  * 기본 디렉터리 권한 `777`에서 `022`를 빼면, 디렉터리 권한은 `755` (rwxr-xr-x).
* **가장 안전한 `umask` 설정**: `umask 077`

  * 파일 권한: `600` (rw-------)
  * 디렉터리 권한: `700` (rwx------)

#### **`umask` 명령어 사용 예시:**

* `umask 077`: 새로 생성된 파일에 대해 읽기/쓰기 권한만 부여하고 실행 권한은 부여하지 않습니다.
* `umask 022`: 새로 생성된 파일에 대해 읽기/쓰기 권한을 소유자에게 부여하고, 그룹과 기타 사용자에게는 읽기 권한만 부여합니다.

### **디스크 쿼터 관리**

디스크 쿼터는 사용자가 할당된 디스크 용량을 초과하지 않도록 제한하는 시스템 기능입니다.

#### **명령어:**

1. **`quota`**: 사용자의 디스크 사용량 및 할당된 쿼터를 확인하는 명령어

   * 사용자의 디스크 쿼터 정보를 출력합니다.

2. **`edquota`**: 사용자 또는 그룹의 디스크 쿼터를 편집할 때 사용하는 명령어

   * `edquota 이름`: 지정된 사용자의 디스크 쿼터를 편집합니다.
   * `edquota -p 이름1 이름2`: 사용자 `이름1`의 쿼터를 사용자 `이름2`에 복사하여 설정합니다.
   * 기본적으로 **vi** 편집기가 열립니다.

3. **`setquota`**: 명령어를 사용해 디스크 쿼터를 설정하는 명령어

   * `setquota -u 이름1 [soft block][hard block][inode soft][inode hard]`

     * **soft block**: 사용자가 초과할 수 있는 용량 (경고가 발생하지만 제한은 없음).
     * **hard block**: 사용자가 더 이상 사용할 수 없는 용량 (강제로 제한).
     * **inode**는 파일 수에 대한 제한입니다.

4. **`repquota`**: 시스템 전체의 디스크 쿼터 정보를 출력하여 보고서를 제공합니다.

5. **`xfsquota`**: XFS 파일 시스템에서 사용자 디스크 사용량을 제한하는 명령어.

   * 예시: `xfsquota -x -c limit bsoft=100m bhard=100m 이름 /home`

     * **bsoft**: 소프트 블록 쿼터 (100MB)
     * **bhard**: 하드 블록 쿼터 (100MB)

#### **쿼터 관련 설정 예시:**

* **`setquota -u user1 50000 100000 0 0 /home`**: 사용자 `user1`에 대해 soft block 50MB, hard block 100MB를 설정.
* **`repquota /home`**: `/home` 디렉터리의 쿼터 상태를 확인.

이 단위들은 디스크 용량과 관련된 크기 단위로 사용되며, 주로 대형 서버나 데이터 센터에서 데이터를 처리하는 데 유용합니다.

### **디스크 및 파일 시스템 관리 명령어**

1. **`df`** - 디스크 용량 확인

   * 시스템에 마운트된 하드디스크 및 파티션의 사용량과 남은 용량을 보여줍니다.
   * 주요 옵션:

     * `-h`: 사람이 읽을 수 있는 형태로 출력 (예: KB, MB, GB 등으로 변환)
     * `-T`: 파일 시스템 종류를 확인 (예: ext4, xfs, etc.)
     * `-i`: inode 사용량을 확인 (파일 시스템에서 사용된 inode 수)
   * 예시:

     ```bash
     df -h
     ```

2. **`du`** - 디스크 사용량 확인

   * 디렉터리별로 파일이나 디렉터리가 차지하는 디스크 용량을 확인할 때 사용됩니다.
   * 주요 옵션:

     * `-s`: 각 디렉터리의 총합만 표시
     * `-h`: 사람이 읽을 수 있는 형태로 출력
   * 예시:

     ```bash
     du -sh /*  # / 이하의 각 디렉터리 크기 출력
     ```

3. **`fdisk`** - 파티션 관리

   * 파티션을 생성, 삭제, 수정하는 데 사용되는 명령어입니다.
   * 주요 옵션:

     * `-l`: 현재 시스템의 디스크 파티션 테이블 정보를 출력
     * `-t`: 파일 시스템 유형 설정
     * `-v`: fdisk 버전 정보 확인
   * 예시:

     ```bash
     fdisk -l
     ```

4. **`fsck`** - 파일 시스템 검사

   * 파일 시스템을 검사하고 필요 시 복구하는 명령어입니다.
   * 주요 옵션:

     * `-s`: 동작 시리얼화
     * `-f`: 강제 점검
     * `-j`: ext3 파일 시스템 점검
   * 예시:

     ```bash
     fsck -f /dev/sda1  # 강제로 /dev/sda1 파일 시스템 점검
     ```

5. **`e2fsck`** - ext2/ext3/ext4 파일 시스템 점검

   * `fsck`와 비슷하지만, 주로 ext2/3/4 파일 시스템에 사용됩니다.
   * 주요 옵션:

     * `-b`: 블록 사이즈 지정
   * 예시:

     ```bash
     e2fsck -f /dev/sda1  # 강제로 ext2/ext3/ext4 파일 시스템 점검
     ```

6. **`xfs_repair`** - XFS 파일 시스템 복구

   * XFS 파일 시스템에서 문제를 진단하고 복구하는 명령어입니다.
   * 예시:

     ```bash
     xfs_repair /dev/sda1  # XFS 파일 시스템 복구
     ```

7. **`lsblk`** - 디스크 블록 장치 정보 확인

   * 시스템에 연결된 블록 장치의 정보를 트리 구조로 보여줍니다.
   * 예시:

     ```bash
     lsblk
     ```

8. **`blkid`** - 블록 장치의 UUID 및 타입 확인

   * 디스크에 할당된 고유한 UUID 및 파일 시스템 타입을 확인하는 명령어입니다.
   * 예시:

     ```bash
     blkid
     ```

### **파일 권한과 특수 권한**

1. **`Set-UID`** (Set User ID)

   * Set-UID가 설정된 프로그램은, 실행 중에 해당 파일의 소유자 권한을 임시로 가져옵니다. 일반 사용자가 해당 파일을 실행하면 root 권한을 얻을 수 있습니다.
   * 예시: `/usr/bin/passwd`는 Set-UID 권한이 설정되어 있어, 일반 사용자도 root 권한으로 비밀번호를 변경할 수 있습니다.

2. **`Set-GID`** (Set Group ID)

   * Set-GID가 설정된 파일은 실행 시 해당 파일이 속한 그룹 권한을 임시로 사용합니다.
   * 이 권한은 주로 그룹 소유자에게만 특정 권한을 부여할 때 사용됩니다.

3. **특수 권한 파일**

   * **`/usr/bin/passwd`**: 일반 사용자가 root 권한으로 실행할 수 있는 대표적인 파일로, Set-UID 권한이 설정되어 있습니다.

### **도커(Docker)**

* **Docker**는 경량화된 가상화 기술로, 하이퍼바이저나 게스트 운영체제 없이 서버 운영에 필요한 프로그램과 라이브러리만 격리하여 설치하고 실행할 수 있습니다. 즉, 애플리케이션 및 그 의존성들이 컨테이너라는 격리된 환경에서 실행됩니다.

#### **Docker의 주요 장점:**

1. **경량화**: 하드웨어 자원을 더 적게 사용하여 효율적으로 운영 가능합니다.
2. **빠른 배포**: 애플리케이션 및 서비스를 신속하게 배포하고 관리할 수 있습니다.
3. **이식성**: 도커 컨테이너는 어느 환경에서든 실행이 가능하므로, 개발, 테스트, 프로덕션 환경에서 일관성을 유지할 수 있습니다.

#### **도커 사용 예시:**

* **컨테이너 이미지 실행**:

  ```bash
  docker run -d -p 80:80 nginx  # nginx 이미지를 기반으로 컨테이너 실행
  ```

* **이미지 빌드**:

  ```bash
  docker build -t my_image .  # 현재 디렉터리에서 Dockerfile을 사용하여 이미지를 빌드
  ```

* **컨테이너 목록 확인**:

  ```bash
  docker ps  # 현재 실행 중인 컨테이너 목록
  ```

### **가상화 및 컨테이너 관련 기술**

1. **XEN, KVM 등으로 나누어 운영되는 가상화 환경**

   * XEN, KVM은 하이퍼바이저 기반의 가상화 기술로 여러 가상 머신을 운영할 수 있게 해주는 기술입니다.

2. **네트워크 장치명**

   * `docker0`: **br**은 포함되지 않음

3. **컨테이너 (Container)**

   * 우리가 구동하려는 애플리케이션을 실행할 수 있는 환경까지 감싸서, 어디서든 쉽게 실행할 수 있도록 해주는 기술입니다.

4. **컨테이너 런타임**

   * 컨테이너를 다루는 도구로, 가장 유명한 것이 \*\*도커(Docker)\*\*입니다.

5. **쿠버네티스 (Kubernetes)**

   * 구글에 의해 설계되었고 현재는 리눅스 재단에 의해 관리되는 컨테이너 관리 프로그램입니다.
   * 여러 서버(노드)에 컨테이너를 분산해서 배치하거나, 문제가 생긴 컨테이너를 교체하거나, 컨테이너가 사용할 비밀번호나 환경 설정을 관리하고 주입해 주는 일을 합니다. 이를 **컨테이너 오케스트레이션**이라고도 합니다.

6. **앤서블 (Ansible)** ✅

   * 여러 개의 서버를 효율적으로 관리할 수 있게 해주는 **환경 구성 자동화 도구**입니다.
   * 2012년 **마이클 데한**이라는 개발자가 만든 오픈소스 소프트웨어로, 2015년부터 **레드헷**이 인수하여 운영 중입니다.
   * **플레이북**을 통해 서버 설정을 자동으로 배포할 수 있게 해줍니다.

7. **오픈스택 (OpenStack)** ✅

   * **계산, 저장소, 네트워크 자원**을 관리하는 **클라우드 운영체제**입니다.
   * 현재 **클라우드 인프라**를 구축할 수 있는 가장 거대한 오픈 소스 프로젝트로, **6개월마다 새로운 버전**을 릴리즈합니다.

### **클라우드 컴퓨팅 모델**

1. **SaaS (Software as a Service)**

   * 클라우드 환경에서 동작하는 응용 프로그램들을 서비스 형태로 제공하는 모델입니다.
   * **공급업체가 모든 서비스를 제어 및 관리**하며, 가장 포괄적인 형태입니다.
   * **예시**: Dropbox, Google Apps

2. **PaaS (Platform as a Service)**

   * 소프트웨어 서비스를 개발하기 위한 **플랫폼**을 제공하는 클라우드 서비스입니다.
   * 개발자는 인프라 영역의 수고를 덜고, **개발에만 집중**할 수 있습니다.
   * **예시**: AWS Elastic Beanstalk, Red Hat OpenShift

3. **IaaS (Infrastructure as a Service)**

   * 서버와 스토리지 같은 IT 하드웨어 자원을 빌려주는 클라우드 서비스입니다.
   * **예시**: AWS, Azure, GCP (퍼블릭 클라우드 업체)

### **네트워크 기술**

1. **Token Ring (IEEE 802.5)**

   * IBM에 의해 개발된 네트워크 기술로, 데이터 전송을 위해 네트워크 채널을 사용하려면 일종의 **사용권을 획득**해야 합니다.

2. **Ethernet (IEEE 802.3)** ✅

   * 데이터 전송(LAN)을 위해 네트워크 채널을 사용하려면 일종의 **사용권을 획득**해야 합니다.
   * **MAC 주소**를 기반으로 상호 간에 데이터를 주고받을 수 있도록 설계된 네트워크 기술입니다.
   * **전송기술**: CSMA/CD
   * **IPV4** → 32bit / **MAC** → 48bit / **IPV6** → 128bit

3. **NFS (Network File System)** ✅

   * **리눅스-리눅스 간 파일 공유** 서비스입니다.

4. **SAMBA** ✅

   * **리눅스-윈도우 간 파일 공유** 서비스로, 초기에는 SMB 프로토콜, 현재는 CIFS 프로토콜을 사용합니다.

5. **FDDI (Fiber Distributed Data Interface)**

   * **광섬유 케이블**을 사용하여 설계된 **링 구조**의 통신망입니다.

#### **네트워크 프로토콜**

1. **ARP** ✅

   * 이더넷 카드에 할당된 **MAC 주소**와 **IP 주소**를 매칭시켜주는 프로토콜입니다.

2. **ICMP** ✅

   * 메시지에 대한 오류 보고와 피드백을 원래 호스트에게 보고하는 역할을 수행하는 프로토콜입니다.

3. **X.25**

   * WAN 상에서 널리 쓰이는 프로토콜입니다. 최근에는 X.25를 기반으로 하는 **Frame Relay**가 등장했습니다.

4. **SMTP** ✅

   * **전자메일 전송 프로토콜**입니다.

5. **IMAP** ✅

   * **전자메일 동기화 프로토콜**입니다.

6. **POP3** ✅

   * **전자메일 수신 프로토콜**입니다.

7. **SNMP** ✅

   * **네트워크 장비 모니터링 프로토콜**입니다.

8. **SMB** ✅

   * **파일 및 프린터 공유 프로토콜**입니다.

9. **DNS**

   * **도메인 이름**과 **IP 주소**를 연결하는 프로토콜입니다.

10. **IRC** ✅

    * **실시간 채팅 프로토콜**입니다.

11. **FTP (20, 21)**

    * **서버/클라이언트 구조**의 서비스로, **TCP/IP**에 의해 제공되는 호스트 간의 파일 전송을 위한 프로토콜입니다.
    * 두 개의 **TCP 연결**을 필요로 하며, 텔넷과 동일하게 **ID**와 **Password**를 이용합니다.
    * **포트 번호**: 20 (전송), 21 (제어)
    * **익명 계정**: 공개 소프트웨어를 제공하는 FTP 서버에 접근할 때 사용할 수 있는 계정입니다. (FTP에서만 가능)

---

#### **FTP 명령어**

1. **hash**

   * 파일 전송 및 다운로드 진행상황을 \*\*#\*\*로 확인할 때 사용합니다.

2. **rename**

   * FTP 서버에 접속하여 원격지의 파일 이름을 변경할 때 사용합니다.

3. **delete**

   * FTP 서버에 접속된 상태에서 파일을 삭제할 때 사용합니다.

4. **open**

   * 다른 FTP 서버에 접속할 때 사용하는 명령입니다. (`/close`)

5. **mget**

   * 로컬 시스템에 여러 개의 파일을 가져올 때 사용합니다. (`/put`)

6. **bye, exit, quit, close**

   * FTP 서버의 연결을 해제하고 셸 프롬프트로 빠져나오는 명령입니다.

#### **Telnet (포트 23)**

* **Telnet**은 원격으로 컴퓨터 시스템에 로그인하고 해당 컴퓨터의 자원을 사용하거나 관리할 수 있는 **네트워크 프로토콜**입니다.
* **포트 번호**가 나오면 텔넷인 경우가 많습니다.
* 리눅스 시스템에서 **inetd 기반으로 운영되는 서비스**입니다.

##### **사용 예시**

##### **관련 메시지 파일**

* `/etc/motd`, `/etc/issue`, `/etc/issue.net`

##### **보안 취약점**

* 보안에 취약하며 원격 접속을 **SSH**로 변경하는 것이 권장됩니다.

* **패키지명**: `telnet-server`

#### **netstat**

* **netstat**: 소켓의 **PID** 및 **프로그램명**, **포트번호**를 출력합니다.
* 라우팅 테이블 정보, IP 주소 기반의 접속 목록 출력, 시스템에 설정된 **게이트웨이** 주소 확인 등의 기능을 제공합니다.
* **Kernel IP routing table**: 커널의 라우팅 테이블 정보를 표시합니다.

---

#### **netstat 명령어 옵션**

* **anp**: 모든 소켓의 PID 및 프로그램명을 출력하고, 호스트명 및 포트명은 숫자값으로 출력합니다.
* **r**: 라우터 테이블 정보를 확인할 때 사용하는 옵션입니다.
* **a**: 모든 소켓 표시
* **n**: IP 주소와 포트 번호를 숫자값으로 표시
* **p**: 지정된 프로토콜의 통계치를 나타냄

---

#### **SS**

* **SS**: 네트워크가 연결된 상태 및 소켓 정보를 보여줍니다.
* **-a**: 모든 소켓 표시
* **-l**: 접속 대기 상태 소켓 표시
* **-n**: DNS가 아닌 IP 주소로 표시
* **-t**: 모든 TCP 소켓 표시
* **-u**: 모든 UDP 소켓 표시
* **-x**: UNIX 도메인 소켓 표시
* **-s**: 각 프로토콜의 통계 정보 표

---

#### **netstat 명령의 상태 값** ✅

* **ESTABLISHED**: 서버와 클라이언트가 서로 연결된 상태를 나타내는 상태 값
* **TIME\_WAIT**: 패킷 처리는 끝났지만 분실 되었을지 모를 느린 세그먼트를 위해 소켓을 열어놓은 상태
* **SYN\_RECEIVED**: 아직 클라이언트에게 확인 메시지를 받지 않은 상태
* **SYN\_SENT**: 로컬 시스템의 클라이언트 애플리케이션이 원격 호스트에 연결을 요청한 상태
* **LISTEN**: 소켓을 열고 기다리는 상태

#### **브로드캐스트 주소 값 구하기** ✅

* **IP 주소**: `192.168.3.129`
* **서브넷 마스크**: `255.255.255.192`

1. 서브넷 부분 255는 제외하고 192와 IP 주소 129를 체크하고 **AND 연산**을 사용
2. 129 → `10000001`
3. 192 → `11000000`
4. **AND 연산시**: `10000000` (128)
5. **결과**: `128`은 B클래스의 최대값인 `191`이 정답

* **정답**: `192.168.3.191`

### **IP 주소 클래스**

---

#### **A 클래스**

* **이진수**: 0xxxxxxx.xxxxxxxx.xxxxxxxx.xxxxxxx
* **십진수**: 0 \~ 127 (128개)
* **IP 주소 범위**: 0.0.0.0 \~ 127.255.255.255

#### **B 클래스**

* **이진수**: 10xxxxxx.xxxxxxxx.xxxxxxxx.xxxxxxx
* **십진수**: 128 \~ 191 (64개)
* **IP 주소 범위**: 128.0.0.0 \~ 191.255.255.255

#### **C 클래스**

* **이진수**: 110xxxxx.xxxxxxxx.xxxxxxxx.xxxxxxx
* **십진수**: 192 \~ 223 (32개)
* **IP 주소 범위**: 192.0.0.0 \~ 223.255.255.255

---

#### **루프백 (loopback)**

* **IP 주소**: `127.0.0.1`
* **설명**: 네트워크 통신에서 패킷이 외부로 전송되지 않고, 송신한 호스트에서 다시 수신한 것처럼 처리되는 주소

  * 패킷의 목적지 IP를 `127.0.0.1`로 설정하면, 패킷은 외부로 전송되지 않고 다시 로컬 시스템으로 돌아가게 된다.

### **클러스터 종류**

---

#### **고계산용 클러스터 (HPC: High Performance Computing Cluster)**

* **설명**: 고성능의 계산 능력을 제공하기 위해 여러 대의 노드를 이용하여 하나의 작업을 병렬로 수행하는 구조
* **다른 이름**: 베어울프 클러스터 (Beowulf)
* **예시**: 슈퍼컴퓨터, 과학 연구용 고성능 컴퓨터

#### **부하 분산 클러스터 (LVS: Linux Virtual Server)**

* **설명**: 서버 접근하는 이용자가 많을 때 부하를 분산하여 서비스 제공
* **용도**: 여러 서버에서 서비스를 제공하여 부하 분산(로드 밸런싱)을 가능하게 함
* **예시**: 웹 서버에서 주로 사용

#### **고가용성 클러스터 (HA: High Availability Cluster)** ✅

* **설명**: 지속적인 서비스를 제공하기 위해 사용하는 클러스터
* **특징**: LVS 클러스터의 단점을 보완
* **동작 방식**: Load Balancer 백업을 만들어 메인에서 이상 발생 시 백업으로 서비스 전환
* **예시**: 웹 서버에서 주로 사용
