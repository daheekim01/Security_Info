# vi 편집기

## 개요

* 유닉스 계열 시스템에서 가장 많이 사용되는 편집기.
* 1976년 빌 조이에 의해 개발됨.

---

## ex 모드 명령어

* **`%d`** : 파일 안의 모든 내용을 지우기 위한 ex 모드 명령어.

* **`%s/windows/linux/g`** : "windows" 문자열을 모두 "linux"로 치환.

* **예시**:

  * `1,7s/fail/success/g` : 1번부터 7번까지의 줄에서 "fail"을 "success"로 치환.
  * `%s/^linux/Linux/` : 파일 전체에서 "linux"로 시작하는 줄을 "Linux"로 치환.
  * ` %s/￦linux/Linux/` : 줄의 끝부분에서 "linux"를 "Linux"로 치환.
  * ` %s/<\linux>/Linux/` : "linux"라는 단어를 "Linux"로 치환.

* **예시 2**:

  * `1,9 %s/문자열/변경할 문자열` : 전체 9줄을 "변경할 문자열"로 치환.

---

## 자동 치환 설정

* **`ab uz uzoogom`** : "uz"라는 문자열을 입력 시 자동으로 "uzoogom"으로 치환되도록 설정.

  * 이 설정은 `.exrc` 파일에 추가해야 함.

---

## vi 편집기 설정 파일

* **`.exrc`** : vi 편집기의 환경 설정을 등록하여, 실행 시 계속적으로 지정된 설정을 적용.
* **`.vimrc`** : vi 편집기 실행 시, 자동으로 행 번호를 표시하도록 설정하는 파일.

---

## vi 명령어 및 설정

* **`vi +15`** 또는 **`vi -ic 15`** : 파일을 불러올 때 커서를 15번째 줄로 이동.

  * 예시: `vi +line.txt` : 파일을 열면서 커서가 마지막 줄로 이동.

* **`set nu`** : 문서의 각 라인 번호를 표시.

  * 각 행 앞에 행 번호가 나타나도록 설정.
  * 일반적으로 `set` 명령을 사용할 때는 `:` (콜론)을 사용하지 않음.

* **`set ai`** : 위 줄과 같은 열에 커서를 위치할 때 지정하는 환경 설정.

* **`set sm`** : 괄호 강조.

* **`set ts`** : 탭 간격 조정.

* **`set list`** : 개행 문자 (`$`), TAB 문자 (`^|`) 등을 확인할 수 있도록 설정.

---

## 파일 복구

* **`r`** : 비정상적으로 종료된 후 작업 중이던 파일을 불러오는 명령.

  * `.swp` 파일의 내용을 불러옴.

---

## 편집 모드 전환

* **`l`, `o`, `a`** : 입력 모드로 전환하는 명령.

  * **`I`** : 커서 위치에서 편집 모드로 전환.
  * **`o`** : 다음 줄로 넘어가서 편집 모드로 전환.
  * **`a`** : 한 칸 뒤로 넘어가서 편집 모드로 전환.

---

## 커서 이동

* **`vi $ 파일명`** : 커서를 줄의 맨 끝으로 이동.
* **`vi - 파일명`** : 커서를 앞줄의 첫 문자로 이동.

---

##  vi 편집기

###  vi 모드

* **입력 모드 (Insert Mode)**: 텍스트를 입력할 수 있는 모드.
* **명령 모드 (Normal Mode)**: 기본적으로 vi를 실행할 때 시작되는 모드로, 커서 이동과 명령을 수행할 수 있음.
* **ex 모드 (Edit Mode)**: 파일을 저장하거나 종료하는 등의 명령을 입력하는 모드.
* **작업 모드**: 사용되지 않는 용어.

---

### vi 명령어

* **`3,9s/^#`**: 3번째 줄부터 9번째 줄까지 주석을 제거. (주석은 `#`으로 시작)

---

### 입력 모드 명령어

* **`a`**: 커서 뒤부터 입력.
* **`A`**: 줄의 마지막에 입력.
* **`i`**: 현재 문자 앞에 입력.
* **`I`**: 줄의 시작 부분에 입력.
* **`o`**: 현재 줄의 다음 줄에 입력.
* **`O`**: 커서가 있는 줄 위에 입력.
* **`x`**: 커서 위치한 곳의 한 문자 삭제.
* **`dd`**: 한 줄 삭제.
* **`dw`**: 커서 위치에서 오른쪽 한 단어 삭제.
* **`db`**: 커서 위치에서 왼쪽 한 단어 삭제.
* **`h`**: 왼쪽으로 이동.
* **`j`**: 아래로 이동.
* **`k`**: 위로 이동.
* **`l`**: 오른쪽으로 이동.
* **`yy`**: 복사.
* **`p`**: 붙여넣기.

## vim 편집기

### 개요

* **vim** (Vi IMproved): **브람 무레나르**가 vi 편집기와 호환되면서 다양한 기능을 추가하여 만든 편집기.

  * **주요 특징**:

    * 패턴 검색 시 하이라이트 기능 제공 (빠른 검색).
    * ex 모드에서 히스토리 기능을 제공.
    * **`set fileencoding=euc-kr`**: vim에서 한글이 제대로 입력되지 않을 경우 사용하는 명령어.
    * 편집 시 다양한 색상을 이용하여 가시성을 높일 수 있음.
    * 정규 표현식 문법과 강력한 문법 강조 기능 제공.

---

## pico 편집기

### 개요

* **pico** (by **아보일 카사르**): 간단하고 직관적인 텍스트 편집기.

  * 터미널에서 실행되며 사용법이 간단함.
  * 다양한 텍스트 편집 기능을 제공하지만, vi나 vim보다 기능이 제한적임.

---

## PICO 편집기

* **개요**:

  * 워싱턴 대학에서 만든 유닉스용 편집기.
  * 윈도우의 메모장처럼 간편하게 사용하도록 설계됨.
  * 모드가 따로 존재하지 않으며, 바로 텍스트 입력이 가능.

* **특징**:

  * **라이선스 문제**: 리눅스 초기에는 대부분의 배포판에 포함되어 있었지만, 최근에는 라이선스 문제로 인해 포함되지 않음.
  * **소스 수정 불가**: 자유 소프트웨어 라이선스가 아니어서 소스 코드 수정이 불가능.
  * **단축키 설명 표시**: 에디터 화면 아래에 단축키에 대한 설명이 표시됨.
  * **배포**: Pine이라는 E-mail 클라이언트 프로그램과 함께 배포됨.

* **주요 단축키**:

  * **`Ctrl + k`**: 한 줄 삭제.
  * **`Ctrl + u`**: 붙여넣기.
  * **`Ctrl + o`**: 편집된 내용 저장.
  * **`Ctrl + a`**: 커서를 현재 줄의 시작 부분으로 이동 (반대는 `Ctrl + e`).
  * **`Ctrl + x`**: 종료.

---

## Nano 편집기

* **개요**:

  * **Pico**의 복제 버전으로, **GNU 프로젝트**에서 개발됨.
  * **Pico**와 동일한 단축키를 제공하며, 편집 화면에 단축키 설명이 정리되어 있는 형태로 나타남.

* **특징**:

  * **Pico 복제**: `Nano`는 `Pico`의 복제 버전으로, `Pico`의 단축키와 유사하게 동작함.
  * **GNU 프로젝트**에서 개발된 무료 소프트웨어.

---

## Emacs 편집기

* **개요**:

  * **리처드 스톨만**이 개발한 매크로 기능이 있는 텍스트 교정 및 편집기.
  * 다양한 편집 기능을 제공하며, 사용자가 자신에게 맞는 모드를 설정할 수 있음.

* **특징**:

  * **다양한 기능 제공**: C, LISP, FORTRAN, HTML 등의 소스 코드 편집을 위한 모드를 지원.
  * **강력한 질의 및 치환 명령**: 문서 편집 및 프로그래밍에 유용한 기능을 갖추고 있음.
  * **LISP 기반 설정**: 환경 설정 언어로 LISP을 사용하여, 사용자가 자신만의 편집 환경을 설정할 수 있음.

* **용도**:

  * 간단한 문서 편집부터 프로그래밍, 메일 송수신까지 다양한 용도로 사용됨.

    * `PID`: 프로세스 ID
  * **우선순위 변경 규칙**:

    * **양수값**: 우선순위를 **낮춤**.
    * **음수값**: 우선순위를 **높임**.

* **`pstree`**

* **`jobs`**

    * 작업이 **중지된 상태**
    * **백그라운드로 실행 중인 상태**
    * **변경되었지만 보고되지 않은 상태**

* **`nohup`**
  터미널이 닫혀도 실행 중인 프로세스를 **백그라운드 프로세스**로 실행할 때 사용.

  * **예시**: `nohup <command> &`

* **`bg`**
  현재 실행 중인 프로세스를 **백그라운드 작업**으로 전환.

  * **`fg`**: 백그라운드 작업을 **포그라운드**로 전환.
  * **예시**: `bg 1`, `fg 1`: `1`번 작업을 백그라운드 또는 포그라운드로 전환.

* **`&`**
  명령어 뒤에 `&`를 붙이면 **백그라운드**에서 작업을 실행.

  * **예시**: `bash script.sh &`: `script.sh`를 백그라운드에서 실행.

# 리눅스 데몬, CRON, 시그널

## 데몬 (Daemon)

* **데몬**: 리눅스 시스템이 부팅 시 자동으로 실행되는 **백그라운드 프로세스**.

  * **주요 특징**:

    * 주기적이고 지속적인 **서비스 요청 처리**를 위해 항상 실행됩니다.
    * 일반 사용자는 이 프로세스를 볼 수 없으며, **루트 권한**만 필요합니다.
  * **종류**:

    * **Standalone**: 메모리에 항상 상주하여 실행되는 방식.

      * **단점**: 메모리 점유로 인한 서버 부하가 큼.
    * **항상 상주하지 않음**: 요청이 들어오면 관련 프로세스를 실행하고 요청이 끝나면 종료됩니다.
* **부팅 시 데몬 실행 방식**:

  * **CentOS 7 이전**: `init` 시스템
  * **CentOS 7 이후**: `systemd` 시스템

## CRON (주기적 작업 자동화)

* **cron**: 주기적으로 반복되는 일을 **자동적으로 실행**하는 서비스.

  * **crontab**: 사용자의 **개별 crontab 파일**을 관리하는 명령어.
* **주요 명령어**:

  * `crontab -e`: **crontab**을 작성하거나 수정
  * `crontab -u <user> -e`: 특정 사용자의 **crontab**을 수정
  * `crontab -r`: 사용자의 **crontab** 삭제
  * `crontab -l`: 현재 사용자의 **crontab 내용 출력**
* **CRON 날짜 형식** (순서):

  ```
  분   // 시   // 일   // 월   // 요일
  ```

  * **요일**: `일(0), 월(1), 화(2), 수(3), 목(4), 금(5), 토(6)` → 0부터 6까지 번호로 지정.
* **주기적인 작업 스케줄링 명령어**:

  * `crontab`, `at`: 주기적인 작업이나 지정된 시간에 실행되는 작업을 설정하는 명령어.

## Kill 시그널

* **시그널 (Signal)**: 프로세스에 특정 작업을 요청하는 메커니즘.

  * **SIGHUP**:

    * **기능**: 프로세스를 **재시작**하거나 터미널과의 연결이 끊겼을 때 발생.
    * **시그널 번호**: 1

  * **SIGINT**:

    * **기능**: 프로세스를 **종료**시키며, 보통 **Ctrl+C**로 발생.
    * **시그널 번호**: 2

# 시그널, 우선순위 및 환경설정 파일

## 시그널 (Signal)

* **SIGQUIT**:

  * **기능**: 프로세스를 종료하고 **코어 덤프**를 생성.
  * **기본 동작**: `Ctrl + \`
  * **시그널 번호**: 3

* **SIGKILL**:

  * **기능**: 프로세스를 **강제로 종료**.
  * **기본 동작**: `kill -9`
  * **시그널 번호**: 9

* **SIGTERM**:

  * **기능**: **정상 종료**를 시도하는 시그널.
  * **기본 동작**: `kill` 명령어로 보내는 기본 시그널
  * **시그널 번호**: 15

* **SIGTSTP**:

  * **기능**: 프로세스를 **대기 상태로 전환**.
  * **기본 동작**: `Ctrl + Z`
  * **시그널 번호**: 20

* **예시**:

  * `kill -1 %2`: **jobs** 명령으로 출력되는 2번 작업에 `SIGHUP` 시그널을 보내어 종료 신호를 보냄.

* **killall**:

  * **기능**: PID 대신 **프로세스명**을 사용하여 해당 프로세스를 종료.

## 프로세스 우선순위

* **PRI (Priority)**:

  * **설명**: 커널이 사용하는 프로세스의 우선순위. 운영체제가 실제로 참고하는 우선순위 값.

* **NI (Nice Value)**:

  * **설명**: 사용자가 설정할 수 있는 프로세스의 우선순위 항목.
  * **효과**: NI 값이 PRI 값에 영향을 미침.

* **우선순위 확인 명령어**:

  * `ps`: 현재 로그인에서 사용 중인 프로세스의 NI 및 PRI 값을 확인할 수 있는 명령어.

## 환경설정 파일

* **`~/.bashrc`**:

  * **설명**: **개인 사용자 환경 설정** 파일. 주로 alias, 함수 등을 설정.

* **`~/.bash_profile`**:

  * **설명**: **로그인 쉘 환경 설정** 파일. 주로 환경 변수, 프로그램 실행 등을 설정.

* **`/etc/bashrc`**:

  * **설명**: **시스템 전체의 bash 설정** 파일. 전역적인 환경 설정.

* **`/etc/shells`**:

  * **설명**: 리눅스 시스템에서 사용 가능한 **셸 정보**가 저장된 파일.

* **`/etc/passwd`**:

  * **설명**: 시스템의 **사용자 정보**가 저장된 파일. 사용자의 계정 정보, UID, GID 등이 포함됨.

## 리눅스 시스템 파일 및 설정

### 로그인 셸 확인

* **명령어**:
  `echo $SHELL`

  * **기능**: 현재 로그인한 사용자의 기본 셸을 확인.

### 시스템 사용자 정보 관련 파일

* **`/etc/shadow`**:

  * **설명**: 시스템 사용자들의 **비밀번호 정보 및 비밀번호 정책**을 저장.
  * **특징**: 일반 사용자는 이 파일을 **볼 수 없음**.

* **`/etc/group`**:

  * **설명**: **로컬 그룹에 대한 정보** 저장. 각 그룹의 이름, GID, 사용자 목록 등.

### 파일시스템 정보 관련 파일

* **`/etc/fstab`**:

  * **설명**: 시스템의 **파일시스템 정보**를 저장.
  * **구성**: 6개의 필드로 구성되어 있음.

    1. **파일시스템 장치**
    2. **마운트 포인트**
    3. **파일시스템 종류**
    4. **옵션**
    5. **덤프** (백업 유무)
    6. **파일점검 옵션** (fsck와 관련)
  * **옵션**:

    * `a`: 파일 시스템을 마운트할 때 사용하는 옵션.
    * `user`: 일반 사용자가 파일 시스템을 마운트할 수 있는 권한 부여.
    *  파일 마운트란 파일 파티션에 디렉토리 형태로 접근하는 것을 의미
* **디스크 쿼터 설정 옵션**:

  * **옵션**:

    * `quota`, `uquota`, `usrquota`, `userquota` 등의 값을 사용하여 디스크 쿼터 설정.

### 부팅 및 환경 설정 파일

* **`/etc/inittab`**:

  * **설명**: 시스템 **부팅 과정 및 초기화 설정**에 관련된 파일.
  * **특징**: 부팅 시 **기본적인 실행 프로세스** 및 X 윈도우 설정을 관리.

* **`/etc/profile`**:

  * **설명**: 시스템의 **환경 변수**와 **시작 프로그램**을 설정하는 파일.
  * **특징**: 모든 사용자에게 반영되는 시스템 전체 설정 파일.

* **`/etc/profile.d`**:

  * **설명**: 여러 응용 프로그램들이 시작될 때 필요한 **스크립트**들이 저장되는 디렉터리.
  * **특징**: 시스템 전역의 환경 변수 설정을 보완하는 스크립트들.

* **`/etc/hosts`**:

  * **설명**: **IP 주소**와 **도메인 이름**을 매핑하는 파일.
  * **기능**: 네트워크에서 도메인 이름을 IP 주소로 변환하는 로컬 설정을 포함.

* **`/etc/sysconfig/network-scripts`**:

  * **설명**: **네트워크 인터페이스 환경 설정**과 관련된 파일들이 저장된 디렉터리.
  * **파일 예시**:

    * `ifcfg-eth0`: 이더넷 인터페이스의 설정 파일.
    * `ifcfg-lo`: 로컬 루프백 인터페이스 설정 파일.

* **`/etc/resolv.conf`**:

  * **설명**: **DNS 서버 주소**를 설정하는 파일.
  * **기능**: 시스템이 DNS 요청을 보낼 서버의 IP 주소를 지정.

* **`/etc/services`**:

  * **설명**: **프로토콜의 포트번호**를 확인할 수 있는 파일.
  * **기능**: TCP, UDP와 같은 프로토콜별로 사용되는 포트 번호와 서비스 이름을 매핑.

### 사용자 설정 관련 명령어

* **`chsh`** (Change Shell):

  * **기능**: 사용자의 로그인 셸을 변경할 때 사용.
  * **옵션**:

    1. `chsh -l`: 변경 가능한 쉘 목록을 출력.
    2. `chsh -s [쉘] [사용자명]`: 사용자의 로그인 쉘을 변경.
       예) `chsh -s /bin/bash username`
    3. `chsh -u`: `chsh` 명령어에 대한 **도움말** 출력.

### 쉘 종류

* **`csh` (C Shell)**:

  * **기원**: 버클리 대학에서 개발.
  * **특징**: C 언어를 기반으로 한 쉘. 사용법이 C 언어와 유사.

* **`ksh` (Korn Shell)**:

  * **기원**: AT\&T의 데이비드 콘에 의해 개발.
  * **특징**: 본쉘(Bourne Shell)을 확장하여 개발되었으며, 많은 고급 기능을 제공.

* **`bash` (Bourne Again Shell)**:

  * **기원**: 본쉘(Bourne Shell)의 개선된 버전.
  * **특징**: 리눅스와 유닉스 시스템에서 가장 많이 사용되는 쉘로, 명령어 자동완성, 명령 히스토리 등 많은 기능을 제공.

## 쉘 종류 및 관련 명령어

### 쉘 종류

* **`csh` (C Shell)**:

  * **기원**: 1978년, 버클리 대학에서 개발.
  * **특징**: C 언어의 문법을 모방한 쉘.

* **`bash` (Bourne Again Shell)**:

  * **기원**: 본쉘(Bourne Shell)을 개선하여, GNU 프로젝트에 의해 개발.
  * **특징**: 가장 널리 사용되는 쉘로, 명령어 자동완성, 히스토리 기능 등 다양한 기능을 제공.

* **`tcsh` (TENEX C Shell)**:

  * **기원**: 1981년, 캔 그리어가 개발.
  * **특징**: C 쉘을 확장하여, 명령행 완성 기능을 통합.

* **`dash` (Debian Almquist Shell)**:

  * **기원**: 더 가볍고 빠른 쉘을 목표로 개발된 쉘.
  * **특징**: 시스템 스크립트에서 빠른 실행을 제공, `/bin/sh`의 대체 쉘로 사용.

### 가장 최근에 등장한 쉘은?

* **`dash`**:

  * **답**: \*\*가장 최근에 등장한 쉘은 `dash`\*\*입니다.

### 쉘 관련 명령어

* **`history`**:

  * **기능**: 이전에 실행한 명령어들을 출력하는 명령어.
  * **예시**: `history 5` → 최근에 실행한 5개의 명령어 출력.

* **`set`**:

  * **기능**: 셸 변수나 환경 변수를 확인하거나 설정할 때 사용.
  * **사용 예시**: 사용자가 로그인한 후 선언한 모든 셸 변수를 확인할 수 있음.

* **`eject`**:

  * **기능**: CD-ROM, DVD-ROM 등의 미디어 장치를 꺼내는 명령어.
  * **사용 예시**: `eject` → CD/DVD 드라이브의 미디어를 꺼냄.

* **`mount`**:

  * **기능**: 파일 시스템을 특정 디렉터리에 마운트하는 명령어.
  * **예시**:

    ```bash
    mount -t xfs /dev/sdb1 /backup
    ```

    * **설명**: `/dev/sdb1` 디바이스를 `/backup` 디렉터리에 XFS 파일 시스템으로 마운트.

* **`umount`**:

  * **기능**: 파일 시스템을 마운트 해제하는 명령어.
  * **예시**:

    ```bash
    umount -a
    ```

    * **설명**: `/etc/mtab`에 명시된 모든 파일 시스템을 마운트 해제. 만약 해제할 수 없는 경우 "device is busy" 메시지가 출력됨.

### 관련 파일

* **`/etc/mtab`**:

  * **기능**: 현재 마운트된 파일 시스템에 대한 정보를 저장하는 파일. `umount -a` 명령은 이 파일을 참조하여 마운트 해제할 파일 시스템을 결정.

## 파일 시스템 및 디바이스 관련 명령어

### `mke2fs`

* **기능**: 파일 시스템을 생성하고 포맷하는 명령어.
* **주요 옵션**:

  * `-t [파일시스템형식]`: 파일 시스템의 유형을 지정.
    예시: `mke2fs -t ext2 /dev/sdb1` → `/dev/sdb1` 디바이스를 ext2 파일 시스템으로 포맷.
  * `-j`: ext3 파일 시스템을 생성.
    예시: `mke2fs -j /dev/sdb1` → `/dev/sdb1`을 ext3 파일 시스템으로 포맷.

### `mkfs.xfs`

* **기능**: XFS 파일 시스템을 생성하는 명령어.
* **사용 예시**:

  ```bash
  mkfs.xfs /dev/sdb1
  ```

  → `/dev/sdb1`을 XFS 파일 시스템으로 포맷.

### 마운트 관련 필드

1. **File system**: 마운트할 파일 시스템.
2. **Mount point**: 파일 시스템이 마운트될 디렉터리.
3. **Type**: 파일 시스템의 종류 (예: ext4, xfs 등).
4. **Option**: 마운트 옵션 (읽기/쓰기 권한 등).
5. **Dump**: 덤프 유틸리티가 백업할지 여부를 설정하는 필드.
6. **Pass**: `fsck` 명령어로 파일 시스템 점검 시 점검 순서.

## 파일 허가권 및 접근 권한

### `chmod`

* **기능**: 파일이나 디렉터리의 접근 권한을 설정하는 명령어.

* **주요 권한**:

  * **소유자 (owner)**: `u`
  * **그룹 (group)**: `g`
  * **기타 사용자 (others)**: `o`
  * **모든 사용자 (all)**: `a`

* **권한 숫자 표현**:

  * `r` (읽기): 4
  * `w` (쓰기): 2
  * `x` (실행): 1

* **옵션**:

  * `-R`: 디렉터리와 하위 디렉터리까지 적용.

#### 예시:

* `chmod 777 /path/to/file`: 모든 사용자에게 읽기, 쓰기, 실행 권한을 부여.
* `chmod u+x file`: 파일 소유자에게 실행 권한을 추가.
* `chmod -R 755 /path/to/directory`: 해당 디렉터리와 하위 디렉터리들에 대해 `755` 권한을 설정.

네트워크 통신이란 인터넷 상에 존재하는 호스트와 서로 패킷을 주고 받는 것이다. 이때, 
호스트 A에서 B로 패킷을 전달하고 싶다고 생각해 보자. 그러면 A의 네트워크 계층에서 
B의 네트워크 계층으로 패킷을 전송해야 한다. 패킷엔 목적지(B)의 IP 주소가 담겨 있
다. 이 정보를 통해 B의 네트워크 계층에 도달한다면, B의 네트워크 계층은 수신한 패킷
을 상위 계층들로 전달하고, 각 상위 계층은 알아서 패킷을 분해하고 순수 데이터(페이
로드)를 수신할 것이다.
그런데 만약 목적지 IP 주소를 127.0.0.1로 설정하게 되면 A의 네트워크 계층은 이 패킷
을 외부로 전송하지 않는다. 전송하지 않고 고스란히 자신이 다시 받은 것처럼 처리하여, 
상위 계층으로 올려 보낸다. 즉 자신이 송신한 패킷을 그대로 수신한 효과를 준다.
